# SceneChunk-OC-LOD
开发世界九宫格加载+8叉数动态显示+LOD的动态加载
## 项目简介

SceneModule 是一个高性能的开放世界场景管理系统，采用九宫格（3x3网格）切分方案，为大世界游戏提供完整的场景加载、卸载和内存管理解决方案。

该系统结合了 LOD（Level of Detail）系统和八叉树（Octree）空间分割技术，实现高效的场景流加载和对象管理，可支持大规模开放世界场景。

## 核心功能

### 🌍 九宫格世界系统
- **动态加载卸载**：基于玩家位置的相对路径九宫格动态加载/卸载相邻场景
- **Scene节点管理**：支持多层级的场景节点结构，灵活应对复杂的世界设计
- **世界边界处理**：智能处理世界边界的场景加载逻辑
- **异步加载机制**：非阻塞式的场景加载，保证游戏流畅性

### 📊 LOD系统（细节级别）
- **自动LOD生成**：GameLODGroup 支持对GameObject的 LOD Renderers 自动分组管理
- **批量转换工具**：LODGroupBatchConverterEditor 支持批量转换现有LOD结构
- **性能优化**：根据摄像机距离动态调整渲染精度
- **对象池管理**：GameLODObjectPool 高效管理LOD对象的回收与复用

### 🔲 空间分割系统（OC - Octree/八叉树）
- **八叉树结构**：OCTree 实现高效的3D空间层级划分
- **高效查询**：快速的空间范围查询（Range Query）和碰撞检测
- **动态更新**：支持运行时动态添加/移除场景对象
- **节点优化**：OCNode 和 OCItem 的层级管理确保查询效率

### 🎯 AOI系统（Area of Interest）
- **兴趣区域管理**：AOIManager 维护玩家的可见范围内的实体列表
- **事件驱动**：进入/离开兴趣区域时触发相应事件
- **网络优化**：减少网络消息传递，只同步玩家可见对象

### 🏃 Marathon模式支持
- 特定游戏模式的场景加载和管理逻辑


## 系统架构

### 分层设计

```
┌─────────────────────────────────┐
│      应用层（游戏逻辑）          │
├─────────────────────────────────┤
│   九宫格世界加载系统 (World)    │
├─────────────────────────────────┤
│   空间查询层 (OC - 八叉树)      │
├─────────────────────────────────┤
│   LOD系统 / 对象池              │
├─────────────────────────────────┤
│      Unity场景与渲染系统         │
└─────────────────────────────────┘
```

### 工作流程

1. **初始化阶段**
   - 初始化WorldLoad，读取世界配置
   - 构建OCTree空间分割结构
   - 初始化GameLODManager管理LOD资源

2. **运行时加载阶段**
   - 监听玩家位置变化
   - 计算当前九宫格位置
   - 加载周围8个相邻Scene
   - 卸载超出范围的Scene

3. **场景管理阶段**
   - OCTree动态维护当前加载的场景对象
   - AOIManager维护玩家视野范围内的对象
   - GameLODManager根据距离调整LOD等级

4. **内存优化阶段**
   - GameLODObjectPool回收未使用的对象
   - 卸载距离过远的Scene及其资源

## 关键类说明

| 类名 | 功能描述 |
|------|--------|
| `WorldLoad` | 世界加载主入口，管理九宫格Scene的生命周期 |
| `WorldNode` | 单个Scene的包装，管理其加载状态和内容 |
| `OCTree` | 八叉树数据结构，提供高效的空间查询 |
| `OCScene` | 场景对象的空间结构管理 |
| `AOIManager` | 管理玩家兴趣区域，高效同步可见对象 |
| `GameLODManager` | LOD系统全局管理器，控制动态细节调整 |
| `GameLODObjectPool` | 对象复用池，减少GC压力 |


## 性能考虑

### 内存管理
- 采用对象池模式减少GC压力
- 远距离场景卸载释放内存
- LOD系统降低远距离渲染负担

### 查询优化
- 八叉树提供O(log n)的空间查询性能
- AOI系统避免全场景遍历
- 缓存机制减少重复计算

### 加载优化
- 异步加载保证帧率稳定（需集成异步接口）
- 优先加载中心场景，延迟加载外围场景
- 预加载策略优化玩家体验
